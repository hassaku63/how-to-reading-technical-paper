## 使用方法

### 実装特化アプローチ
1. **Phase 0**: 学習目標とリソースの明確化
2. **Phase 1**: 実装可能な要素の特定
3. **Phase 2**: 具体的な実装計画の立案

### 段階的実装
- MVP（最小実装版）から開始
- 各段階で動作確認
- 機能を段階的に追加

## 実装戦略

### MVP アプローチ
- 最小限の機能で動作する版を作成
- 段階的に機能を追加
- 早期のフィードバック取得

### テスト駆動開発
- テストを先に書く
- 実装しながらテストを実行
- 品質を保ちながら開発

## 関連ツール

- `identifyImplementables` - 実装可能要素特定ツール
- `evaluateComplexity` - 実装難易度評価ツール

## 注意事項

- 実装可能性を常に意識する
- 現在のスキルレベルに応じた計画を立てる
- 段階的アプローチで挫折を防ぐ
